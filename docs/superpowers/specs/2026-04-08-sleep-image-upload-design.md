# WebUI Sleep画面アップロード機能 設計書

Issue: https://github.com/zrn-ns/crosspoint-jp/issues/28

## 概要

WebUIに独立した「Sleep」ページを追加し、E-inkデバイスのスリープ画面用画像をブラウザからアップロードできるようにする。画像のクロップ・色調整・ディザリング処理は全てブラウザ側（JavaScript）で完結させ、ESP32のRAMを消費しない設計とする。

## 対象デバイス

- X4: 800 × 480 px
- X3: 792 × 528 px
- デバイスの画面解像度は `/api/status` から取得し、クロップ領域を動的に決定する

## ページ構成

`/sleep` エンドポイントで配信する独立ページ（FontsPageと同列）。

### 初期表示（一覧画面）

- ナビゲーションバーに「Sleep」タブを追加
- アップロードエリア: `input[type=file]` を `accept="image/*"` でラップしたドロップゾーン（タップで写真選択/カメラ撮影）
- 保存済み画像一覧: `/sleep/` ディレクトリ内のBMPファイルをサムネイルグリッド表示
  - 各サムネイルにファイル名、サイズ、削除ボタンを表示
  - サムネイルはBMPファイルをそのまま `<img>` で表示（ブラウザがBMPを描画可能）

### 編集画面（クロップ＆調整）

画像選択後に表示。ダーク背景で編集に集中できるUI。

**クロップUI:**
- Canvas上に画像を描画し、`transform`（translate + scale）で移動・拡大
- クロップ領域 = Canvas全体 = デバイス解像度アスペクト比に固定
- 三分割グリッドオーバーレイ表示
- タッチ操作: 1本指ドラッグ（移動）、2本指ピンチ（拡大縮小）
- マウス操作: ドラッグ（移動）、ホイール（ズーム）
- ズームスライダー（補助操作）

**画像調整:**
- 明るさスライダー（-100 〜 +100）
- コントラストスライダー（-100 〜 +100）
- リアルタイムでプレビューに反映

**ディザリング方式選択:**
- Atkinson（デフォルト）: 誤差を6方向に1/8ずつ拡散。E-ink向き、シャープな仕上がり
- Floyd-Steinberg: 誤差を4方向に重み付き拡散。滑らかなグラデーション
- しきい値: 4段階量子化（0-63→黒, 64-127→濃灰, 128-191→薄灰, 192-255→白）。イラスト・ロゴ向き

**プレビュー:**
- 4階調（黒 #000000 / 濃灰 #555555 / 薄灰 #AAAAAA / 白 #FFFFFF）でシミュレーション表示
- 調整パラメータ変更時にリアルタイム更新

**アクションボタン:**
- 「キャンセル」: 一覧画面に戻る
- 「アップロード」: BMP生成 → WebSocket転送

## 画像処理パイプライン（ブラウザ側）

```
input[type=file]
  → FileReader.readAsDataURL()
  → new Image() に読み込み
  → Canvas に描画（クロップ: translate + scale で範囲決定）
  → getImageData() で RGBA ピクセルデータ取得
  → グレースケール変換（ITU-R BT.601: 0.299R + 0.587G + 0.114B）
  → 明るさ・コントラスト適用
  → ディザリング適用（選択方式で4階調に量子化）
  → 8bit パレットBMP バイナリ生成
  → WebSocket チャンク転送
```

### BMP生成仕様

- フォーマット: 8bit パレットBMP（Windows BITMAPINFOHEADER）
- パレット: 4エントリ（0x00, 0x55, 0xAA, 0xFF）、残り252エントリはゼロ埋め
- ピクセルデータ: ボトムアップ、行は4バイト境界にパディング
- ファイルサイズ: 14（ファイルヘッダー）+ 40（DIBヘッダー）+ 1024（パレット）+ width × height = 約385KB（800×480）

## ESP32側の変更

### 新規エンドポイント

| メソッド | パス | 目的 |
|----------|------|------|
| GET | `/sleep` | SleepPage.html 配信 |
| GET | `/api/sleep/images` | `/sleep/` 内のBMPファイル一覧（JSON配列） |
| GET | `/api/sleep/thumbnail?file=xxx` | BMPファイルを直接返す（Content-Type: image/bmp） |
| DELETE | `/api/sleep/delete?file=xxx` | 指定BMPファイル削除 |

### `/api/sleep/images` レスポンス例

```json
[
  {"name": "sleep_001.bmp", "size": 385078},
  {"name": "sleep_002.bmp", "size": 385078}
]
```

### `/api/status` 拡張

既存レスポンスに以下を追加:

```json
{
  "screenWidth": 800,
  "screenHeight": 480
}
```

`renderer.getScreenWidth()` / `renderer.getScreenHeight()` から取得。

### WebSocket転送

既存のフォントアップロードプロトコルをそのまま再利用:

```
クライアント → サーバー: "START:sleep_001.bmp:<size>:/sleep/"
サーバー → クライアント: "READY"
クライアント → サーバー: [バイナリ 4096B] × N回
サーバー → クライアント: "PROGRESS:<received>:<total>"（65KBごと）
サーバー → クライアント: "DONE" または "ERROR:<message>"
```

### ファイル保存先

- ディレクトリ: SDカード上の `/sleep/`（ドットなし）
- 初回アップロード時にディレクトリ自動作成
- ファイル命名: `sleep_001.bmp`, `sleep_002.bmp` ... 自動連番（既存ファイルとの重複回避）

## ファイル構成の変更

| ファイル | 変更種別 | 内容 |
|----------|----------|------|
| `src/network/html/SleepPage.html` | 新規 | Sleep画面管理ページ（HTML/CSS/JS一体） |
| `src/network/CrossPointWebServer.cpp` | 変更 | エンドポイント追加、ハンドラー実装 |

## エラーハンドリング

### ブラウザ側

- 画像読み込み失敗 → エラーメッセージ表示、再選択を促す
- WebSocket接続失敗 → リトライ3回、失敗時はエラー表示
- アップロード中のページ離脱 → `beforeunload` イベントで警告
- 巨大画像（一眼カメラ等の高解像度写真）→ Canvas上でリサイズしてから処理するため問題なし

### ESP32側

- `/sleep/` ディレクトリ未存在 → 初回アップロード時に `mkdir` で自動作成
- SDカード容量不足 → 書き込み失敗検知、`ERROR:disk full` 返却、不完全ファイル削除
- 不正ファイル名 → パストラバーサル防止（`..` 禁止、英数字+`_`+`-`+`.`のみ許可）
- WebSocket切断 → 不完全ファイル自動削除（既存の仕組み）
- 同名ファイル → 自動連番により回避

## セキュリティ

- ファイル名サニタイズ: パストラバーサル攻撃防止。サーバー側で連番命名するため、クライアント指定のファイル名は使用しない
- 保存先固定: `/sleep/` ディレクトリのみ。任意パスへの書き込み不可
- ファイルサイズ上限: 500KB（800×480 8bit BMPは約385KB）
- Content-Type検証: WebSocket STARTコマンドのパス先頭が `/sleep/` であることを確認
